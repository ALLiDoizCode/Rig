Refined Design Decision (2026-02-26)

Selected Strategy: Forgejo Layout Patterns + Rig's Own Visual Identity + Progressive Disclosure

Rig follows Forgejo's layout patterns (flex-list repo display, tab navigation, filter/sort bars, file tree structure) while establishing Rig's own visual identity (color palette, typography, border radius, shadow depth). Same bones, different skin. Forgejo's open-source templates (in forgejo/templates/) and CSS (in forgejo/web_src/css/) serve as layout references. Codeberg (codeberg.org) serves as the live visual reference.

This refines the original Direction 2 (GitHub Evolved) + Direction 5 (Progressive Disclosure) selection. The key change: the primary layout reference shifts from GitHub to Forgejo because its source code is available for direct inspection, and its community overlaps with Rig's early adopters. Note: NIP-34 is a Nostr protocol standard for git collaboration — it is not specific to Forgejo/Gitea. Forgejo is chosen as a layout reference purely for its open-source UI quality and community alignment.
